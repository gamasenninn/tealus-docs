# 音声メッセージ

テキストの代わりに音声でメッセージを送信できます。送信された音声は自動で文字起こし・AI整形されます。

## 録音と送信

1. 入力欄の **マイクボタン** をタップ
2. ボトムシートが表示され、録音が開始される
3. 録音中はタイマーと波形が表示される
4. 「送信」をタップして送信、または「キャンセル」で取り消し

!!! tip
    チャット画面はそのまま表示されるため、録音しながらメッセージを確認できます。

## 再生

送信された音声メッセージは吹き出し内に再生バーとして表示されます。

- **▶ ボタン** をタップして再生
- 再生バーで再生位置を変更可能

## 文字起こし（gpt-4o-transcribe + AI整形）

音声メッセージ送信後、以下の処理が自動的に実行されます。

1. **音声文字起こし**: OpenAI の音声認識 API による文字起こし（v0.2.2〜default は `gpt-4o-transcribe`、環境変数 `WHISPER_MODEL` で `whisper-1` / `gpt-4o-mini-transcribe` への切替可、「処理中...」と表示）
2. **AI整形**: 文字起こし結果を読みやすい文章に整形（「AIが文章を整えています...」と表示）

整形された文字起こしテキストは、音声の吹き出しの下に表示されます。

## 文字起こしの編集

文字起こし結果を手動で修正できます。

1. 自分の音声メッセージの「文字起こしを編集」ボタンをタップ
2. テキストエリアで内容を修正
3. 確定ボタンを押す

編集した内容はリアルタイムで全メンバーに反映されます。

### 編集履歴

過去の編集内容は「履歴ボタン」からバージョン管理で確認できます。

!!! note "編集権限"
    デフォルトでは送信者のみ編集可能ですが、ルーム設定で「メンバー全員が編集可」に変更できます。

## 再文字起こし

文字起こしが失敗した（`status=error`）場合や、結果が不適切な場合に、**手動で retry** できます（v0.2.2〜、[#216](https://github.com/gamasenninn/tealus/issues/216)）。

1. 自分の音声メッセージの「再文字起こし」ボタンをタップ
2. 確認ダイアログで OK
3. 新しいバージョンの文字起こしが生成され、Socket.IO 経由でリアルタイムに UI が更新される

| voice の状態 | ボタンの位置 |
|---|---|
| `status=error`（失敗） | prominent（目立つ位置に配置） |
| `status=done`（成功済） | 「編集」「履歴」と並ぶ |

!!! note "新 version として保存"
    再文字起こしは編集と同じく**新しい version を作成**するため、過去の結果は履歴ボタンから確認できます（破壊的に上書きされません）。

## 連続再生

複数の音声メッセージを連続して再生できます。

1. ルーム設定画面 → 「個人設定」→「音声の連続再生」をONにする
2. 音声メッセージを再生すると、再生完了後に次の音声メッセージが自動で再生される
3. 再生中のメッセージは自動スクロールで画面中央に表示される

停止するには、再生中の一時停止ボタンを押してください。

## AI回答の読み上げ（TTS）

AIエージェントの回答を、トランシーバーから音声で自動読み上げできます。

### 有効化

1. プロフィール設定を開く
2. 「AI回答の読み上げ」をONにする

### 動作の流れ

1. メッセージを送信すると、トランシーバーが**自動接続**される
2. AIの回答がテキストで表示されると同時に、**トランシーバーから音声が流れる**
3. 30秒間新しい回答がなければ、トランシーバーが**自動切断**される（読み上げ中は切断されません）

!!! tip
    画面を見なくてもAIの回答を聞けるため、作業中のハンズフリー運用に便利です。

読み上げの音声モデル（声の種類）は、管理者がダッシュボードのルーム設定から変更できます。詳しくは [TTS 読み上げ](../advanced/tts.md) を参照してください。

## 音声リプライ

音声メッセージでもリプライ（引用返信）が可能です。リプライされた音声メッセージには、引用元のテキスト（文字起こし含む）が表示されます。
