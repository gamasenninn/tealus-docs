# TODO機能

チャットのメッセージをTODOリストとして管理できます。

## TODOの追加

1. メッセージを**長押し**（PCでは右クリック）
2. コンテキストメニューから「TODO」を選択
3. TODOタグを選択、または新しいタグを作成

## TODOの管理

検索ページのTODOフィルタから、TODOの一覧を管理できます。

| 操作 | 説明 |
|---|---|
| チェックボックス | 完了/未完了の切替 |
| 優先度（★） | 優先度の設定 |

完了したTODOは取り消し線と薄い色で表示されます。

!!! tip
    TODOは既存のタグシステムを利用しているため、タグでの分類・絞り込みが可能です。

## AI による TODO 操作

UI 操作だけでなく、AI（[Light / Deep agent](../../dev/agents.md)）にも TODO の管理を依頼できます。AI は MCP ツール経由で検索・完了マークを自律的に行います（[#197](https://github.com/gamasenninn/tealus/issues/197)）。

### 完了マーク

AI に「このタスク完了でマークして」「○○の件、完了として」と依頼すると、`mark_tag_done` ツールでメッセージのタグの `is_done` 状態が切り替わります。UI の チェックボックス操作と同じ結果になります。

### 未完了一覧の取得

「未完了の TODO を教えて」「やり残しを一覧して」のような依頼に対し、AI は `search_messages(tag_names="TODO", is_done=false)` で未完了タスクのみを抽出して返します。検索パラメータの詳細は [メッセージ検索](search.md) の「AI による検索（自然言語）」セクションを参照してください。

### 手動操作との併用

手動マーク（UI のチェックボックス）と AI による完了化は同じデータを更新するため、いずれの方法で完了化しても他方の表示に即座に反映されます。チームで使う場合、人間が UI で操作したものを AI が後から把握することも、AI が一括完了したものを UI で確認することも可能です。

## 任意タグ運用

TODO 機能は内部的に**汎用タグシステム**を利用しており、`TODO` 以外の任意タグも自由に作成・運用できます（[#185](https://github.com/gamasenninn/tealus/issues/185)）。

### ジェネリックな運用例

| タグ名（例） | 用途 |
|---|---|
| `TODO` | 未着手のタスク |
| `重要` | 優先的に確認すべき内容 |
| `検討` | 議論が必要な議題 |
| `完了` | 終わった項目（is_done と独立した運用も可） |
| `日常業務` | ルーチンタスク |
| `tealus関係` `製品X` 等 | プロジェクト・トピック別の分類 |

### 「完了状態」を持つチェックリストとして

`mark_tag_done` は **任意タグの `is_done` 状態切替**に使えるため、TODO に限らず「完了状態を持つチェックリスト」全般として活用できます。

- 「`検討` タグの完了マーク」= 議論が結論に至った印
- 「`重要` タグの完了マーク」= 確認済みの印

タグ命名と運用ルールは利用者の裁量です。Tealus はタグの意味を固定せず、`mark_tag_done` / `search_messages` という primitive のみを提供します。チームの運用に合わせて自由に設計してください。
