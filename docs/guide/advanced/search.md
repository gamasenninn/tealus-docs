# メッセージ検索

トーク内のメッセージをキーワードで検索できます。

## 検索方法

1. ボトムナビゲーションまたはヘッダーの検索アイコンをタップ
2. 検索バーにキーワードを入力
3. 検索結果が表示される

## 検索結果の見方

検索結果には以下の情報が表示されます。

- メッセージの抜粋テキスト（キーワードがハイライト表示）
- ルーム名
- 送信者名
- 送信日時

## メッセージへのジャンプ

検索結果をタップすると、該当メッセージの位置に自動でジャンプします。

## 検索対象

- テキストメッセージ
- 音声メッセージの文字起こしテキスト
- すべての参加ルームが横断検索の対象

## AI による検索（自然言語）

UI の検索バーと同じ検索基盤を、AI（[Light / Deep agent](../../dev/agents.md)）も MCP ツール `search_messages` 経由で利用できます（[#194](https://github.com/gamasenninn/tealus/issues/194)）。「先週の議論をまとめて」のような自然言語で AI に依頼すると、AI が適切な検索パラメータを組み立てて結果を返します。

### AI への話しかけ方

ジェネリックな依頼例と、AI が組み立てる検索パラメータの対応です。

| 話しかけ方の例 | AI が組み立てるパラメータ |
|---|---|
| 「先週の議論をまとめて」 | `since` / `until` を直近 7 日間にセット |
| 「未完了の TODO を教えて」 | `tag_names="TODO"` + `is_done=false` |
| 「○○について話していたメッセージを探して」 | `q="○○"` |
| 「△△さんが先月言っていたことを思い出して」 | `sender_id=△△` + `since` を先月初日 |
| 「画像だけまとめて」 | `type="image"` |
| 「`tealus関係` タグで完了済みのもの」 | `tag_names="tealus関係"` + `is_done=true` |

!!! info "narrowing filter は最低 1 つ必要"
    `q` / `room_id` / `sender_id` / `since` / `tag_names` / `type` のうち最低 1 つを指定する設計です。AI も自然言語の文脈からどれか 1 つは導出してくれますが、抽象的すぎる依頼（「全部教えて」など）には narrowing を促し返してきます。

### snippet 索引パターン

`search_messages` の結果には各メッセージの **snippet（抜粋）**が含まれます。AI はこれを「索引」として扱い、必要な詳細だけ後から取得する 2 段階アクセスが推奨パターンです。

```
ユーザー: 「先週の TTS 議論まとめて」
     ↓
AI: search_messages(q="TTS", since="先週", limit=10)
     ↓ snippet を 10 件取得 = 索引
AI: 「以下が候補です。詳細が必要なものを指定してください」
     ↓ ユーザーが選んだら
AI: get_messages(message_id) で本文を展開
     ↓
AI: ユーザーに要約 / 詳細回答
```

snippet だけで要約できる場合は本文取得をスキップすることで、コンテキスト消費を抑えながら効率的に議論を辿れます。

### タグ運用

タグは **user が自由に命名** できます。システム固定タグはありません（[#185](https://github.com/gamasenninn/tealus/issues/185)）。

- 例: `TODO` / `tealus関係` / `重要` / `検討` / `完了` / `日常業務` …
- `tag_names` パラメータは CSV で **AND 検索**（例: `tag_names="TODO,important"` で両方付いたメッセージのみ）
- `is_done` で完了状態フィルタ（`tag_names` 指定時のみ有効）

!!! tip "API は意味を固定せず primitive を提供"
    Tealus はタグ自体に「TODO」のような特別な意味を持たせず、汎用的な「メッセージに付けられる文字列ラベル」として提供します。タグ命名と運用ルールは利用者の裁量に委ねられており、AI もそのタグ体系に追従して検索・更新します（詳細は [TODO機能](todo.md) の「AI による TODO 操作」セクションを参照）。
